# 变更日志工作流索引

本仓库用根目录 [`CHANGELOG.md`](CHANGELOG.md) 记录**可对外说明的变更**（按日期分段、追加写入）。每次完成一批修改并准备 **commit** 时，应把与本次提交相关的说明**同步写进** `CHANGELOG.md`，再与代码一起提交。

**分支提交哈希归档**：[`COMMIT_LOG.md`](COMMIT_LOG.md) 按「分支 → 日期 → 提交」整理，与 `CHANGELOG.md` 互补；面向可追溯的 commit 列表而非产品说明。
## 何时写入

- 行为、接口、配置、依赖、构建或用户可见文案有变化时：**必须**追加条目。
- 仅格式化、注释、无行为差异的重命名：可按团队习惯决定是否记录；若记录，写一句即可。

## 写在哪里

- 在 `CHANGELOG.md` 顶部的 **`## YYYY-MM-DD`**（当天日期）下增加小节。
- 若当天已有同名日期标题，在同一日期下追加新的小节（`### 标题`），不要新开一个重复日期。
- **只追加**，不删改历史条目；纠错在**最新**条目中说明（见 [`AGENTS.md`](AGENTS.md)「变更日志规范」）。

## 建议结构

- 小节标题：简短说明主题（如「某某功能修复」「某某 API 调整」）。
- 条目：用 `-` 列出改了什么；涉及文件时用仓库内相对路径或现有 `CHANGELOG.md` 中的链接风格，与历史条目保持一致。

## 与 commit 的对应关系

- 一次 commit 可以对应 `CHANGELOG.md` 里的一条或多条 bullet；也可以把同一主题的多个文件改动合并成一个小节。
- 若一次 MR/PR 包含多个独立主题，建议在 `CHANGELOG.md` 里分成多个 `###` 小节，便于日后检索。

## 自动化助手（Agent）说明

在 [`AGENTS.md`](AGENTS.md) 中约定了助手在完成修改时同步维护 `CHANGELOG.md` 的条件；人类贡献者与 Agent 共用同一套规则与同一文件。
